# SkillSwap 🔄

SkillSwap is a real-time, gamified collaborative platform designed to facilitate peer-to-peer learning. Users can offer their expertise, connect with others to learn new skills, schedule live video sessions, maintain learning streaks, and track comprehensive session outcomes.

**Live Demo:** [https://skillswap-connect.netlify.app/](https://skillswap-connect.netlify.app/)

## ✨ Key Features

- **🚀 Modern SaaS Dashboard**: A visually stunning, fast, and responsive user interface displaying analytics, pending requests, and upcoming sessions.
- **🤝 Learn-by-Doing Marketplace (Task Challenges)**: A dedicated hub where users can post specific technical challenges and accept tasks from peers.
- **📈 Gamification System**: Dynamic XP tracking, level progression, continuous learning streaks, and accuracy metrics to incentivize engagement.
- **📹 Real-Time Video Sessions**: Peer-to-peer 1-on-1 video collaboration powered by the Jitsi React SDK.
- **💬 Live Chat**: Embedded direct messaging functionality backed by robust WebSockets.
- **📅 Session Outcome Board**: An intuitive log for users to leave verified session reflections, ratings, and accurately track skills both taught and learned.
- **🔐 Secure Authentication**: Fast, stateless JWT-based authentication combined with secure password hashing.

## 🛠️ Tech Stack

### Frontend (Client)
- **Framework**: React 19 (Hooks, React Router)
- **Styling**: Tailwind CSS & Vanilla CSS (custom design system, glassmorphism, dynamic animations)
- **Data Visualization**: Recharts
- **Video Conferencing**: `@jitsi/react-sdk`
- **Real-Time Data**: `socket.io-client`
- **Networking & Auth Tools**: `axios`, `jwt-decode`

### Backend (Server)
- **Runtime**: Node.js
- **Framework**: Express.js
- **Database**: MongoDB (managed via `mongoose`)
- **Real-Time Engine**: `socket.io`
- **Security**: `jsonwebtoken` (JWT), `bcryptjs`
- **Middleware**: `cors`, `dotenv`

## 💡 Implementation Highlights & Interview Prep

If you are asked about the technical decisions and architecture of this platform in an interview, here are the most important talking points:

### 1. Gamification Engine (State & XP Calculation)
- **The Challenge**: Updating user progress, accuracy, and levels continuously without heavy backend processing.
- **The Solution**: The `User` database model inherently stores current XP, Streaks, Level, and Total Attempts. Whenever a challenge is completed or a session outcome is logged, the Express backend updates the exact XP increment securely. It evaluates XP thresholds dynamically to level up the user immediately without requiring complex background CRON jobs.

### 2. Live Video Integration (WebRTC via Jitsi)
- **The Challenge**: Building an embedded, low-latency, and high-fidelity video conferencing system.
- **The Solution**: Constructing raw WebRTC from scratch requires deploying complex signaling servers and STUN/TURN servers to bypass NAT barriers. By integrating the `@jitsi/react-sdk`, the platform handles advanced peer-to-peer video communication off-the-shelf. It acts as an embedded IFrame meeting node generated from the scheduled session attributes.

### 3. Real-Time Interactions (Socket.io)
- **The Challenge**: Enabling instant chat messaging and real-time collaboration request notifications across different clients.
- **The Solution**: A persistent WebSocket connection is initialized simultaneously with the user session. The frontend extensively relies on `useEffect` hooks to manage connection state. **Crucial detail:** the architecture ensures strict component cleanup logic (e.g., `socket.disconnect()` or clearing event listeners on unmount) to prevent severe memory leaks and duplicated message renders.

### 4. Stateless Session Authorization (JWT)
- **The Challenge**: Creating a fast and highly secure API access layout without maintaining sessions in server memory.
- **The Solution**: Upon login, a JWT payload encapsulating the user ID is generated by the server and signed with a `.env` secret. The client stores it locally. To prevent unnecessary backend calls, the frontend leverages `jwt-decode` to extract expiration states and selectively un-render protected React Router paths if the token is invalid. 

### 5. Session Outcome Board (Data Modularity)
- **The Challenge**: Coupling interactive collaborative sessions with verified feedback.
- **The Solution**: The database separates a `Session` model from a `SessionOutcome` model. Once a session completes, users log an outcome referencing the specific `sessionId`. This tracks confidence levels (1-5), immediate reflections, and next steps, rendering perfectly on the frontend within a dynamic Accordion-based outcome board for user accountability. 

### 6. Modern Light-Themed SaaS UI Architecture
- **The Challenge**: Maintaining a cohesive, aesthetic component system.
- **The Solution**: Tailwind CSS handles utility-first styling but is paired alongside global structural variables in `index.css`. This controls the tokenized "Primary", "Accent", and "Surface" layers uniformly. Complex animations like `animate-fade-in` and nuanced hover micro-interactions deliver a premium, lightweight SaaS UX.

## 🚀 Running the Project Locally

1. **Clone the Repository** and navigate to the project root.
2. **Install Dependencies**:
   - For the client: `cd client && npm install`
   - For the server: `cd server && npm install`
3. **Set Up Environments**:
   - Ensure you have a `.env` file in your `server` directory configuring your `MONGO_URI`, `JWT_SECRET`, and `PORT`.
4. **Start the Application**:
   - Run the backend: `cd server && npm run dev` (Runs Nodemon on port 5000 typically).
   - Run up the frontend: `cd client && npm start` (Starts React-scripts on port 3000).
